# 非目標とスコープ制約

## ルール

- Nest.js / Express などのアプリケーションフレームワーク、ORM、Web サーバを本リポジトリに追加しません。本リポジトリは軽量 CLI のみを提供します。
- 対象は現在チェックアウトされているリポジトリと vendoring 済みのツールリポジトリに限定します。他リポジトリの自動変更や横断的な操作は扱いません。
- 秘密情報（トークン、秘密鍵、Webhook URL など）はコード・コミット・ドキュメントに含めません。CLI 引数か環境変数として受け取り、必要に応じて GitHub Actions のシークレットを利用します。
- 既存スクリプトの依存解析・自動マージ判断を超えるロジック（例: PR の自動レビューや自動コンフリクト解消の拡張）は、明示的な依頼がない限り追加しません。

## 理由

このプロジェクトは tiramiss フォーク運用に特化した最小構成として設計されており、機能追加が増えるほど運用フローの障害点が増えます。スコープを絞り、`src/utils/` の薄い共通基盤に依存するスタイルを維持することが、保守性と CI 安定性の両立に不可欠です。

## 適用範囲

- 新規依存パッケージ追加の提案を行う前に、上記スコープに収まるかを確認します。
- 既存スクリプトの「便利機能」追加提案では、本ルールに沿って必要性を再評価します。
