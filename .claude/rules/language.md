# 言語ポリシー

## ルール

LLM エージェントによる応答、コード内コメント、エラーメッセージ、ログ出力、コミットメッセージは原則として日本語で記述します。

## 理由

本リポジトリの保守者は日本語話者であり、Issue・コミット・既存スクリプト内のコメントとログも日本語で統一されています。英語と日本語が混在するとレビュー時の認知負荷が高まり、CI ログから問題箇所を追う際の効率も低下します。

## 適用範囲

- LLM エージェントの対話レスポンス全般。
- 新規追加・編集する `src/` 配下の `.ts` ファイル内コメントと `Error` メッセージ。
- 新規追加するワークフローのステップ名・コメント。
- 新規追加するドキュメント（`.md`）。
- 例外: ライブラリ仕様上の識別子（`UPSTREAM_REMOTE`, `BASE_REF` などの環境変数名）、git コマンド名、ファイルパスは原文のままにします。
